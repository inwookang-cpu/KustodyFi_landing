KIDB Q&A 내부용 치트시트

미팅일: 2026.03.17  |  참석: KIDB (오윤정 대표, 부사장, 정훈 전무) × KustodyFi (강인우 대표)
⚠ 내부 전용 — 외부 공유 금지
미팅에서 실제 나온 질문 & 우려
미팅 중 발언
"우리는 FX에 대해 인가된 거지, 스테이블코인이나 디지털자산에 대해 인가된 건 아니다"
맥락: KIDB가 온체인 NDF 파트너십에 대해 본인들의 인가 범위를 명확히 선 그은 발언.

대응: Stage 1~2는 블록체인과 무관한 순수 데이터 인프라 + 기업 프라이싱임을 재확인. 온체인 NDF 거래소(Stage 3)는 싱가포르 자회사에서 MAS 라이선스로 운영. KIDB는 거래소 운영 주체가 아니라 보고 채널 + 데이터 파트너.

향후: 스테이블코인 관련 인가가 풀리는 시점에 맞춰 다시 논의. 지금은 이 주제를 push하지 않는 게 좋음.
KIDB가 "승인을 받으려면 그건 또 다른 문제"라고 했음. 규제가 선행되어야 한다는 인식. 우리가 먼저 거래소를 만들어서 실수요를 증명한 후 접근하는 순서가 맞음.
미팅 중 발언
"데이터가 퍼블릭에 나가는 게 가능한지는 체크를 해봐야"
맥락: 가격 데이터를 외부 서비스에 노출하는 것에 대한 내부 컴플라이언스 확인 필요.

대응: "가격만 보여주는 거면 가능은 한데"라고도 했으므로 완전 거절은 아님. 고객 정보 마스킹 + 가격 데이터만 활용이면 가능성 열려 있음.

향후: 정훈 전무님과 기술 미팅에서 구체적으로 어떤 데이터가 나갈 수 있는지 범위 협의.
"Powered by KIDB" 표기도 내부 확인 전까지는 강조하지 말 것. 프레젠테이션에 "(표기 여부 협의)"로 이미 수정 완료.
미팅 중 발언
"저희와 뭘 할 수 있는 건 좀 나중에 얘기겠네요. 바로 4월, 5월은 좀 어렵다"
맥락: 단기 액션은 어렵다는 명확한 시그널. 소화할 시간이 필요.

대응: 타임라인을 강요하지 않음. LOI + 정기 경과 공유(월 1회 이메일)로 관계 유지. POC 시점은 AI Agent 준비 완료 후 양사 합의.

향후: 3Q 이후 프로덕트가 나오면 데모와 함께 재접근.
미팅 중 발언
"중개사니까 늘 앞에 나가서 할 필요도 없고, 두 번째 정도로 가는 게"
맥락: 중개사 특성상 선발주자(first mover)가 되는 것을 회피. 업종 자체가 다 같이 움직여야 뭘 할 수 있음.

대응: KIDB를 front에 세우지 않는 구조로 설계. KustodyFi가 기업 시장에서 먼저 움직이고, KIDB는 데이터만 제공하는 역할로 포지셔닝.

향후: 다른 중개사(IP 등)도 같이 움직이면 KIDB 참여 부담이 줄어듦. 멀티소스 전략과 연결.
IP 부사장님도 만났는데 "관심은 있으나 MBP 하고 있는 게 있다"고 함. 조만간 업데이트된 덱으로 재접근 예정.
미팅 중 발언
"API를 만들 수도 없는 게, 우리가 플랫폼이 있어서 가격을 넣는 게 아니라 사람이 말로 하는 거"
맥락: KIDB는 전자 플랫폼이 없음. 보이스 브로킹 기반이라 데이터 피드 API 자체가 존재하지 않음.

대응: KustodyFi가 수신 서비스를 직접 구축. KIDB가 API를 만들 필요 없음. AI Agent가 호가를 수집하는 방식으로 접근. KIDB 측은 AI Agent 도입 시 보안 세팅만 필요.

향후: 정훈 전무님과 구체적인 데이터 수신 프로세스 협의. 내부 시스템에서 취합·배포하는 시점에 데이터를 가져오는 방식 검토.
KIDB도 "안 할 이유가 없어 보이고요"라고 했음. 기술적 허들이 낮아지면 참여 의지는 있는 것으로 판단.
미팅 중 발언
"한국자금중개는 본인들이 직접 하겠다는 입장이었다"
맥락: 우리 대표가 한국자금중개를 만났을 때 MBP로 이해하고 직접 하겠다고 한 사례 공유. KIDB 측은 이를 듣고 반응 관찰.

대응: KIDB에게는 "한국자금중개와 다른 결"임을 강조. 우리는 MBP가 아니라 기업향 프라이싱 + AI Agent. KIDB가 직접 하지 않아도 업무 범위가 확장되는 구조.

향후: 한국자금중개가 직접 만드는 것과 우리 서비스의 차이점을 명확히 정리해둘 것.
사전 준비 Q&A (미팅에서 안 나왔으나 향후 대비)
대비용
"우리 데이터가 기업한테 가면 은행들이 싫어하지 않나?"
이미 KMB/SMBS 웹에서 IRS/CRS 데이터를 공개 중. 기업이 접근 못 하는 게 아니라 해석을 못 하는 것.
오히려 기업이 시장가를 이해하면 거래 의사결정이 빨라져 거래량이 증가 — 은행에도 이득.
대비용
"KustodyFi가 망하면?"
KustodyFi가 AI Agent를 먼저 완성한 후 POC를 시작하므로 KIDB 투입 최소, 비용 제로. Downside가 거의 없는 구조.
POC 시점은 양사 합의로 결정하며, Stage 2 진입도 POC 결과 확인 후 KIDB가 결정.
대비용
"우리한테 직접적인 이득이 뭔데?"
즉시: KIDB 인터뱅크 데이터가 기업 고객에게 도달.
중기: AI Agent로 브로커 업무 효율화 + 기업 사용 데이터로 상품 기획 근거 확보.
장기: 데이터 사업 RS. TP ICAP Parameta는 연 ₩5,000억+ — KIDB도 동일한 데이터 자산을 이미 보유 중.
대비용
"KIDB가 직접 하면 되지, 왜 KustodyFi가 필요한가?"
KIDB의 핵심 역량은 인터뱅크 중개와 데이터. 기업 프라이싱 엔진, AI Agent는 기술 스택이 완전히 다른 영역.
직접 개발하면 1~2년 + 전문 인력 채용이 필요. KustodyFi와 협업하면 AI Agent가 준비된 상태에서 빠르게 기업 시장 진출 — KIDB는 데이터만 제공하고 기업 도달은 KustodyFi가 책임.
대비용
"블록체인, 스테이블코인 — 리스크 아닌가?"
Stage 1~2는 블록체인과 무관. 순수 데이터 인프라 + 기업 프라이싱.
On-chain NDF 거래소는 Stage 3 (싱가포르 자회사), MAS 라이선스 하에서 운영. KIDB는 거래소의 보고 채널 + 데이터 파트너이지, 블록체인 운영 주체가 아님.
대비용
"규제 이슈는 없나? 스타트업과 협력해도 괜찮나?"
Stage 1은 데이터 피드 제공 — 기존 데이터 유통과 동일한 업무 범위 내.
Stage 2도 기업 고객에게 프라이싱 정보를 제공하는 것이므로 규제 이슈 없음.
