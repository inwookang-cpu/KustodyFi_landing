전통 FX Swap — 은행 없이는 불가능한 구조
기업이 환헤지를 하려면 은행에 전화하고, 은행 마진이 포함된 가격을 받아야 합니다.
만약 이 구조를 은행 없이, On-Chain에서 실행할 수 있다면?

KustodyFi는 전통 FX Swap의 가격 산출 메커니즘(CIP)을 그대로 가져오되,
은행 대신 LP Pool + AMM이 유동성을 제공하는 구조를 만듭니다.
Structure
전통 FX Swap vs KustodyFi Dated Futures

KustodyFi Dated Futures
Base Currency: USD Stablecoin (USDC/USDT/USDE/USD1 등)
Quote Currency: KRW / KRW Stablecoin (sKRW 확장 예정)
금리 Source: USD Stablecoin = RWA/DeFi Oracle, KRW = FX Implied
Daycount: USD Stablecoin = ACT/365, KRW = ACT/365
유동성: Unified LP Pool + AMM (CLOB)
Spread: 5bp one-side (고정 Base) + Dynamic
만기: 셋째주 수요일 (NW + 1~6M, 9M, 12M)
접근성: 지갑만 있으면 누구나
Fee: Maker 0.5bp / Taker 1bp
Settlement: Cash-settled (Stablecoin 차금결제)
핵심 차이: CIP 공식은 동일하되, USD → USD Stablecoin으로 base가 바뀌고, 은행 → LP Pool로 유동성 공급자가 바뀜.
USD Stablecoin ≠ USD. Stablecoin 금리는 별도 oracle (RWA 70% + DeFi 30%)에서 조달. 이것이 새로운 term structure의 출발점.

USD Stablecoin Term Structure — 시장에 없는 것을 만든다
현재 USD Stablecoin에는 term structure가 존재하지 않습니다.
Aave/Compound는 변동 금리(overnight). BUIDL은 단일 수익률.

"3개월 USD Stablecoin 금리는 얼마인가?" — 아무도 답할 수 없음.
KustodyFi dated futures가 체결되면, 각 tenor에서 implied rUSD.S(T)를 역산할 수 있음.
Phases
3-Phase Term Structure Discovery
Oracle Flat → RWA Ladder → Hybrid → Market-Driven. 체결이 쌓이면 oracle 방향이 뒤집힙니다.
Phase 1 Oracle Flat
rUSD.S: RWA/DeFi 블렌딩 (flat)
방향: Oracle → Forward
역할: 초기 호가 산출
전 tenor 동일 rate
Phase 1.5 RWA Ladder
rUSD.S(T): 만기 매칭 RWA
방향: T-bill ladder → rate(T)
역할: Tenor별 rate 차별화
3M T-bill ≠ 1Y T-bill yield
Phase 2 Hybrid Blend
rUSD.S: α×Implied + (1-α)×RWA
방향: 체결가 + RWA Ladder
역할: 시장 데이터 반영
체결마다 implied rate 역산
Phase 3 Market-Driven
rUSD.S: 체결가 TWAP
방향: 체결 → Reference
역할: Term Structure = Market
Oracle 방향 뒤집힘
Phase 1.5: 만기 매칭 RWA로 자체 Term Structure 구성
현재 LP Pool은 BUIDL/USDY 등 단일 RWA에 전액 배치. 향후 잔여 만기가 다른 T-bill tokenized asset에 분산 배치하면,
tenor별로 다른 yield를 반영한 USD Stablecoin term structure를 구성할 수 있음. (예: 3M BUIDL 4.8% vs 1Y OUSG 5.2%)
