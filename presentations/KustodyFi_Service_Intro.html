StableFX

AI가 호출 가능한 Pricing Plane

사람이 가격을 만드는 시장에, 검증 가능한 기준가격을 설정합니다

1

단일 Reference Rate

인터뱅크 기반 기준금리 + Swap Point 이론가 제공. 시장 전체를 관통하는 하나의 기준.

2

맞춤형 가격 계산

기업별 funding cost 입력 → 실제 접근 가능한 이론가 도출. 내 조건에 맞는 가격.

3

Swap vs Spot 비교

전문가 없이도 금융공학적 판단이 가능한 대시보드. 어떤 상품이 유리한지 즉시 확인.

4

프로그래머블 가격

API로 이론가 배포 → AI Agent 연동 ready. 코드로 가격을 읽고, 판단하고, 실행.

StableFX는 '정보비대칭 해소 도구'가 아니라 '프로그래머블한 가격 레이어'입니다.
StableFX CLI

터미널에서 바로 확인하는 Reference Rate

$ stablefx usdkrw USDKRW Spot Trade: 2026-03-21 Value Date: 2026-03-24 Tenor Maturity Days Bid Ask Mid(Val) ===== ========== ===== ========== ========== ========== SPOT 2026-03-24 3 1,485.30 1,485.40 1,485.40
Slack bot 연동: /stablefx usdkrw

Available Commands

  • stablefx usdkrw — mid price
  • stablefx usdkrw fwd 1M — forward outright
  • stablefx usdkrw termstructure — swap points term
  • stablefx usdkrw full — full structure + implied yields
  • stablefx usd df 4/16 — discount factor
  • stablefx krw zrate 4/16 — zero rate
SEAL

AI Agent가 실행해도 되는 상태를 만드는 Control Plane

권한 · 한도 · 승인 · 감사 증적이 코드에 내장돼야 AI에게 실행을 온전히 맡길 수 있습니다

1

Policy Engine

권한 · 한도 · 라인 자동 검증. Agent/Human 동일 룰 적용. 예외 없는 정책 실행.

2

Audit Trail

거래 전/중/후 감사 로그. 타임스탬프 · 승인자 · 변경이력. 완전한 추적 가능성.

3

Dual Approval

Human-in-the-Loop 다단계 승인. AI 자율실행 + 사람 최종 승인. 신뢰와 효율의 균형.

4

Fail-closed

기본값 = 차단. 허용은 명시적 승인만. Break-glass 비상 프로토콜 탑재.

Pain Point — Buy-side

기업이 겪는 FX 거래의 구조적 문제

기준가격 통일 불가

은행마다 기준이 다르고, 이론가를 알 수 없음

비딩 비교 불가

시간/매체 제각각, 감사 대응 복잡

비효율적 거래

Swap 대신 Spot 매도+매수 따로 실행

전문인력 수급 어려움

금융공학적 판단 자체가 불가능

Solution → StableFX 단일 Reference Rate + Chat 동시 비딩 + 최적 상품 가이드

브로커가 겪는 운영 리스크

라인 관리 지옥

Solution → SEAL 실시간 라인 관리 + Chat 타임스탬프/큐 관리

Pain Point — Sell-side

은행이 겪는 비효율과 리스크

동시다발적 고객 대응 어려움

기준 가격 불일치

Margin에 대한 소모적 논쟁

라인체크 리스크

거래 후 재확인 → 책임 이슈 발생

Settlement 인력 과다

Solution → Chat 주문 큐 + StableFX 스프레드 가시화 + SEAL 정산 자동화
