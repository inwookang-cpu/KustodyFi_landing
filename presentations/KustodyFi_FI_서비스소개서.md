# KustodyFi — 금융기관(FI) 향 서비스소개서

은행 sell-side(영업·딜러)가 귀행 거래 데이터만으로 고객사를 입체로 보고, 관계 이탈을 먼저 잡고, 신규 기회를 더 빨리 발견하는 FI Cockpit — 데이터 주권·비자문·감사 보증을 채택 신뢰축으로 삼는다.

---

## 1. 은행 sell-side를 위한 KustodyFi — FI Cockpit

**핵심:** 고객사를 한 화면에서 입체로 보고, 귀행 거래 데이터만으로 관계를 키운다.

- FI Cockpit의 중심은 Counterparty 360(CP360) — 업종·본사 위치·담당자, 딜러별 거래량·건수·거래스타일, 거래량 추세를 한 화면에 모은다.
- 거래량 감소를 즉시 캐치하는 관계 이탈 조기경보 — Funded/AutoDebit mandate·Layered Margin·4-eyes까지 영업·딜러 일선에서 바로 본다.
- 데이터 주권이 최우선 신뢰축 — 모든 지표는 귀행 거래에만 한정되어, 타행 거래는 귀행 화면에 합산·노출되지 않으므로 내 경쟁데이터가 새지 않는다.
- 살아있는 데이터 — 분석은 실제 체결 거래에서 자동 파생되고, 고객사가 자가관리하는 프로필은 변경 시 거의 즉시 알림으로 받는다.
- 비자문 포지셔닝(운영·관계 가시화이지 시장판단·매매권유 아님) + RLS 격리 + SEAL 감사 보증. 과금은 Subscription + MNF(비자문 운영서비스), 신규 고객은 born-on-rail.

_목적: 은행 영업·딜러에게 FI Cockpit이 고객사를 입체로 보고 관계를 키우는 도구임을, 데이터 주권을 채택 신뢰축으로 각인시킨다. 문의 sales@kustodyfi.com_

## 2. 문제: 은행 sell-side는 "내 고객"을 제대로 못 본다

**핵심:** 고객사 가시성은 부족하고, 이탈은 뒤늦게 알아채며, 신규 발굴은 비효율적인데, 외부 도구를 쓰자니 경쟁데이터 유출이 두렵다.

- 가시성 부족 — 고객사의 업종·본사 위치·담당자, 딜러별 거래량·건수·거래스타일이 흩어져 있어 한 화면에서 입체적으로 파악하기 어렵다.
- 이탈 늦은 인지 — 거래량이 줄어드는 신호를 실시간으로 잡지 못해, 관계가 식은 뒤에야 알아챈다. 조기경보가 없다.
- 신규 발굴 비효율 — 달러원 매도 수요처럼 귀행에 economic value가 되는 상대를 찾는 작업이 수작업·감(感) 의존으로 흩어진다.
- 경쟁데이터 유출 우려 — 외부 분석 도구에 거래를 올리면 우리 거래가 타행 화면에 비칠까 봐 채택을 망설인다.
- 수기 부담 — 고객 프로필·연락처를 사람이 갱신해야 해 항상 낡고, 실제 체결 데이터와 따로 논다.

_목적: 은행 영업·딜러가 자신의 일상 고통을 즉시 자기 일로 인식하게 하고, 데이터 주권이 채택의 전제 조건임을 각인시킨다._

## 3. 기존 대비 진화: 수동 분석 → 자동 분석 → 능동 추천

**핵심:** 과거엔 세일즈가 카운터파티별 거래를 수기로 분석해야 했다. 이제 CP360이 자동으로 분석하고, 향후엔 귀행에 필요한 회사를 능동 추천한다.

- 과거(수동) - 영업·딜러가 카운터파티별 거래 내역을 일일이 들여다보고, 이 고객이 "주로 무엇을 하는지"를 사람이 수기로 분석해야 했다.
- 현재(자동) - CP360이 귀행 실제 체결 거래에서 딜러별 거래량·건수·거래스타일·추세를 자동 파생한다. 분석은 늘 최신이고, 거래량 감소도 자동으로 캐치된다.
- 향후(능동 추천) - 한발 더 나아가, 귀행에 필요한 방향의 거래를 하는 회사를 먼저 추천한다(거래스타일 매칭 → 관계 주선·계좌 개설 지원, 기업 opt-in 전제, 거래 중개 아님).
- 일관된 원칙 - 모든 단계에서 데이터는 귀행 거래에만 한정(타행 비노출), 비자문 포지셔닝 유지.

_목적: "수동 → 자동 → 능동 추천" 진화로 기존 방식 대비 KustodyFi의 도약을 한 장에 각인시킨다. 문의 sales@kustodyfi.com_

## 4. CP360 개요 — 한 화면에서 보는 고객사 360

**핵심:** CP360은 은행 영업·딜러가 한 화면에서 고객사를 입체적으로 파악하는 콕핏 — 업종·본사 위치·담당자·딜러별 거래·증거금까지 모두 귀행 거래에만 한정해 보여준다.

- 한 화면 입체 파악 — 업종, 본사 위치(지도·출장 동선), 담당자 연락처, 딜러별 거래량·건수·거래스타일, Funded/AutoDebit mandate·Layered Margin·4-eyes를 한 콕핏에서 본다.
- 관계 이탈 조기경보 — 거래량 추세를 자동 추적해 감소를 즉시 캐치, 고객이 멀어지기 전에 영업이 먼저 움직인다.
- 데이터 주권(신뢰 1순위) — 모든 지표는 귀행 거래에만 한정. 타행 거래는 귀행 화면에 합산·노출되지 않는다(우리은행 화면에 하나은행 거래가 비치지 않음).
- 살아있는 데이터 — 분석은 실제 체결에서 자동 파생(수기 아님), 프로필은 고객사가 자가관리하고 변경 시 거의 즉시 알림, 연락처는 회사 정보와 RM 비공개 메모를 격리한 하이브리드다.
- 비자문·안전 설계 — 운영·관계 가시화이지 시장판단·매매권유가 아님. RLS 격리로 화면을 분리하고 SEAL이 은행 감사 보증을 제공한다.

_목적: 은행 영업·딜러에게 CP360이 내 경쟁데이터가 새지 않는 안전한 한 화면 고객사 360임을 각인시키고 도입 문의로 이끈다. 문의 sales@kustodyfi.com_

## 5. 데이터 주권 = 신뢰: 귀행 거래만, 타행은 비노출

**핵심:** CP360의 모든 지표는 귀행 거래에만 한정된다 — 우리은행 화면에 하나은행 거래는 비치지 않는다.

- 귀행 거래만 집계 — 타행 거래는 귀행 화면에 합산되거나 노출되지 않는다. 채택의 핵심은 내 경쟁데이터가 새지 않는다는 확신이다.
- RLS 격리로 은행 간 데이터를 구조적으로 분리하고, SEAL이 결정론적 서명·fail-closed 정책·불변 감사기록으로 추출도구가 아닌 감사 보증을 제공한다.
- 살아있는 데이터 — 분석은 귀행 실제 체결에서 자동 파생(수기 아님), 프로필은 고객사 자가관리이며 변경 시 RM에게 거의 즉시 알림. 연락처는 회사 정보와 RM 비공개 메모를 격리한 하이브리드다.
- 향후 확장(프리미엄) — 거래 스타일 매칭으로 귀행에 economic value가 되는 업체를 찾아 관계 주선·계좌 개설 지원(거래 중개 아님, 기업 opt-in 전제). 타행 기밀은 비노출, 마켓플레이스 프로스펙트도 opt-in 집계만.
- 비자문 포지셔닝 유지 — 운영·관계 가시화이지 시장판단·매매권유가 아니다.

_목적: 데이터 주권(귀행 거래만, 타행 비노출)을 FI Cockpit 채택의 최우선 신뢰축으로 각인시킨다. 문의 sales@kustodyfi.com_

## 6. 살아있는 데이터: 자동 파생 + 고객사 자가관리 + 변경 즉시 push

**핵심:** CP360의 모든 지표는 귀행 실제 체결에서 자동으로 살아나고, 고객사가 직접 갱신한 프로필 변경은 거의 즉시 귀행에 알림으로 닿는다.

- 자동 파생 — 거래량·건수·딜러별 거래스타일·추세는 귀행 실제 체결 거래에서 자동 계산되어 수기 입력 의존 없이 항상 최신, 추세 감소도 자동으로 잡힌다.
- 고객사 자가관리 + 즉시 push — 업종·담당자 등 프로필은 고객사가 직접 갱신하고, 변경 발생 시 귀행 영업·딜러에게 거의 즉시 알림이 닿아 RM이 옛 연락처로 헛걸음하지 않는다.
- 데이터 주권 — 이 살아있는 데이터는 전부 귀행 거래에만 한정되어, 타행 거래는 귀행 화면에 합산·노출되지 않아 내 경쟁데이터가 새지 않는다.
- 연락처 하이브리드 — 고객사가 관리하는 회사 공식 정보와 귀행 RM의 비공개 메모를 분리·격리해 공유 정보와 내부 기밀이 섞이지 않는다.
- 갱신 주기 — 기본은 배치 동기화, 실시간 정밀 집계는 프리미엄. 운영·관계 가시화 목적이며 시장판단·매매권유 아닌 비자문 포지셔닝을 유지한다.

_목적: CP360 데이터가 수기가 아니라 귀행 거래에서 자동으로 살아 움직이고 변경이 즉시 닿으면서도 타행 데이터와 철저히 격리됨을 보여, 영업·딜러가 신뢰하고 의존할 운영 신호임을 설득한다. 문의 sales@kustodyfi.com_

## 7. 딜러 행동 분석 + 이탈 조기경보 — 거래가 줄기 전에 먼저 안다

**핵심:** CP360이 딜러별 거래량·건수·거래스타일을 실제 체결에서 자동 파생하고, 거래량 추세 감소를 즉시 캐치해 관계 이탈을 조기에 알린다 — 모두 귀행 거래에만 한정.

- 딜러별 입체 분석 — 거래량·거래건수·거래스타일(달러원 방향성·테너·시간대 패턴)을 딜러 단위로 한 화면에 정리해, 누가 귀행의 핵심 관계인지·어떤 고객이 식어가는지 한눈에 파악한다.
- 이탈 조기경보 — 고객사·딜러별 거래량 추세가 꺾이는 순간 즉시 캐치해 알림. 관계가 완전히 끊기기 전에 RM이 먼저 움직일 수 있는 시간을 확보한다.
- 데이터 주권(신뢰 핵심) — 모든 지표는 귀행 거래에만 한정. 타행 거래는 귀행 화면에 합산·노출되지 않아(우리은행 화면에 하나은행 거래 안 비침) 내 경쟁데이터가 새지 않는다.
- 살아있는 데이터 — 분석은 실제 체결에서 자동 파생(수기 입력 아님), 기본은 배치 집계·실시간 정밀 집계는 프리미엄. RLS 격리와 SEAL 감사 보증으로 운영 안전성을 확보한다.
- 비자문 포지셔닝 — 관계·운영 가시화이지 시장판단·매매권유가 아님. 향후 거래스타일 매칭은 기업 opt-in 전제의 관계 주선·온보딩 facilitation이며 거래 중개가 아니다.

_목적: 딜러·RM이 거래량 감소를 사후가 아니라 사전에 포착해 관계 이탈을 막게 하고, 데이터 주권을 채택 신뢰축으로 각인시킨다. 문의 sales@kustodyfi.com_

## 8. 프리미엄: 거래스타일 매칭으로 관계 주선 + 계좌 개설 지원

**핵심:** CP360의 거래스타일 분석을 한 단계 확장해, 귀행에 필요한 방향의 거래를 하는 업체를 찾아 관계 주선과 계좌 개설을 돕는 신규 발굴 레인 — 거래 중개가 아닌 B2B 소개·온보딩 facilitation이며 기업 opt-in이 전제다.

- 신규 발굴 — 거래스타일 매칭으로 귀행이 원하는 흐름(예: 달러원 매도 수요)을 가진 업체를 식별해 영업·딜러에게 후보로 제시한다.
- Economic value 안내 — 어느 회사를 온보딩하면 귀행에 도움이 되는가를 정성 근거와 함께 안내해, 막연한 신규영업이 아닌 방향성 있는 발굴을 가능케 한다.
- 중개 아님 — KustodyFi가 하는 일은 관계 주선·소개·계좌 개설 지원(온보딩 facilitation)이며, FX 거래 자체를 중개하지 않는다. 비자문 포지셔닝을 유지한다.
- 기업 opt-in 전제 — 소개 대상 기업이 동의한 경우에만 노출하며, 동의 없는 기업은 후보에 오르지 않는다.
- 매칭 범위와 데이터 주권 — 귀행 고객 내부는 자유롭게(L2), 마켓플레이스 프로스펙트는 집계·opt-in 기반(L3 governed)으로. 타행 기밀 거래는 어떤 경우에도 귀행 화면에 노출되지 않는다.

_목적: CP360의 관계 가시화를 신규 관계 창출로 확장하는 프리미엄 가치를 보여주되, 중개가 아닌 opt-in 기반 소개·온보딩 facilitation이며 데이터 주권이 지켜짐을 sell-side 독자에게 각인시킨다. 문의 sales@kustodyfi.com_

## 9. 운영 통제: Funded/AutoDebit mandate · Layered Margin · 4-eyes

**핵심:** 신용 노출과 결제 위험을 사전 통제로 묶어, 딜러는 영업에 집중하고 통제는 시스템이 책임진다.

- Funded/AutoDebit mandate — 고객 사전 펀딩과 자동 출금 권한을 약정 단계에 묶어 결제일 미입금·미결제 위험을 사전 차단한다(신규 born-on-rail 고객 기준, 기존 장부 추출 아님).
- Layered Margin — 계층형 증거금으로 변동성·고객 등급별 완충을 차등 적용해 한도 소진과 손실 노출을 구조적으로 억제한다.
- 4-eyes — 한도·약정·예외 처리에 이중 승인 게이트를 적용해 단독 실수와 우회를 차단하고 감사 가능한 통제선을 유지한다.
- 데이터 주권 — 모든 통제 지표는 귀행 거래에만 한정. 타행 거래는 귀행 화면에 합산·노출되지 않는다(우리은행 화면에 하나은행 거래 안 비침).

## 11. 보안·규제 포지셔닝: 비자문 · 데이터 주권 · 감사 보증

**핵심:** FI Cockpit은 시장판단·매매권유가 아닌 운영·관계 가시화이며, 귀행 데이터는 RLS로 격리되고 SEAL이 모든 행위를 불변 기록으로 감사 보증한다.

- 비자문(non-advisory) 포지셔닝 — CP360은 귀행 운영·관계를 가시화하는 도구이지 시장판단이나 매매권유를 하지 않으며, MNF는 자산운용이 아닌 비자문 운영 서비스 과금이다.
- 데이터 주권(RLS 격리) — 모든 지표는 귀행 거래에만 한정되어, 타행 거래는 귀행 화면에 합산·노출되지 않는다(우리은행 화면에 하나은행 거래 안 비침). 내 경쟁데이터가 새지 않는다.
- 관계 주선은 거래 중개 아님 — 거래스타일 매칭으로 달러원 매도 수요 업체 등을 찾아 소개·계좌 개설을 돕는 B2B 온보딩 facilitation이며, FX 거래 중개가 아니고 반드시 기업 opt-in을 전제로 한다.
- SEAL 감사 보증 — 결정론적 서명 파이프라인과 fail-closed 정책엔진, 불변 감사기록으로 모든 행위를 추적한다. 은행에 추출도구가 아니라 감사 보증으로 작동한다.
- 신규 고객 only(born-on-rail) — 기존 은행 장부 추출·마이그레이션이 아니라 신규 고객이 레일 위에서 시작한다. 기존 장부에 손대지 않아 도입 리스크가 없다.

## 12. 도입 절차 한눈에: 신규 고객부터 바로 시작

**핵심:** 기존 장부 이관 없이 신규 고객(born-on-rail)부터 시작 — 한 번의 셋업으로 귀행 영업·딜링 데스크가 CP360을 바로 쓴다.

_목적: 은행 sell-side에게 신규 고객 중심 4단계 도입 경로와 신뢰축(데이터 주권·비자문)을 제시하고 sales@kustodyfi.com 연결로 전환을 유도한다._
