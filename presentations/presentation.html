SCENARIO 02
자금 스케줄 확인
이번 주 수출대금 입금 + OUTRIGHT SELL 결제 + SWAP 만기 한눈에
일상
박차장은 엑셀 '자금 스케줄표' 관리. 수출대금 입금 예정일, OUTRIGHT SELL 결제일, SWAP 만기일을 날짜별로 정리.
수시
바이어 수출대금 입금 예정 건 + 은행 환전 거래 체결 건을 김대리가 엑셀에 수기 추가/삭제.
금요일
박차장이 다음 주 스케줄 검토. 수출대금 유입 vs 환전 유출 타이밍 맞는지 확인. 안 맞으면 은행에 전화.
⚠ 수출대금 입금 지연 시 환전 거래 결제 자금 부족 위험. 수기로는 파악이 늦음.
⚠ SWAP Near/Far 연결이 끊겨 순 USD 포지션 파악 수작업 필요.
⚠ SEAL(내부 검증) 개념 없음. 바이어 입금 확정인지 불확실한 채로 스케줄에 반영.

SCENARIO 03
Invoice 수신 → 선물환 헷지
해외 원자재 공급업체 Invoice $3M (결제일 3개월 후) — USD를 사야 하는 상황
Invoice 수신
구매팀이 독일 공급업체에서 Invoice $3M 수신. 결제일 3개월 후. 이메일로 자금팀에 전달. "이거 결제해야 해요."
내부 공유
자금팀 김대리가 이메일 확인 후 엑셀 자금 스케줄표에 수기 입력. 구매팀이 안 알려주면 모름. 부서 간 공유가 전화/메일 의존.
D-90~D-1
선물환(Forward)을 하고 싶지만 — 이론가를 모르고, 시스템도 없음. 그냥 환율 지켜보면서 기다림.
D-Day
결제일 당일 SPOT으로 $3M 매수. 3개월 전 1,380이었는데 오늘 1,420. 현물환밖에 못하니 그대로 매수.
결과
$3M × 40원 = ₩1.2억 추가 비용. 헷지했으면 막을 수 있었던 손실.
⚠ 구매팀이 Invoice를 알려줘야만 자금팀이 인지. 누락/지연 빈번.
⚠ 선물환 이론가를 모르니 현물환만 반복. 환율 변동은 그냥 떠안음.
⚠ CFO: "왜 대비 안 했어?" — 대비할 수단 자체가 없었음.
누가 올리든, 어디서든 — PDF만 올리면 끝.
구매팀이든 영업팀이든 Invoice PDF만 시스템에 올리면, 회사 전체 자금 스케줄에 자동 반영.
따로 자금팀에 전화/이메일할 필요 없음. 담당자에게 알림이 자동으로 감.

거래 오류 수정 — 삭제와 취소
$2M을 $20M으로 잘못 입력했을 때
발견
김대리가 $2M을 $20M으로 잘못 입력. 이미 자금 스케줄표에도 반영.
수정
거래대장 수정 + 자금 스케줄표 수정 + 일일현황표 수정본 재발송.
취소
이미 체결된 건 취소 시: 은행 전화 → 취소 → 엑셀 3곳 수정 → 보고서 재발송.
⚠ 엑셀 여러 파일에 동일 데이터 분산. 하나만 수정하면 불일치.
⚠ 수정 이력 없음. 원래 뭐였는지 추적 불가.

SCENARIO 08
Multi-bank RFQ
복수 은행에 동시 호가 요청 — 경쟁 구도만으로 거래 비용 절감 (S/B SWAP: Swap Points 낮을수록 유리)
상황
1M FX SWAP (SELL Near / BUY Far) 거래가 필요. 주거래은행 신한에 전화해서 호가 요청.
호가 수신
신한은행이 Swap Points 4.20을 제시. 이게 적정한 건지 판단할 근거가 없음. "이런 건가 보다" 하고 체결.
비교 불가
다른 은행에 같은 조건으로 호가를 받아보고 싶지만, 은행마다 전화해서 같은 말 반복해야 함. 시간도 부족하고, 관계도 부담.
⚠ 단일 은행에 의존. 경쟁 호가 없이 제시가 곧 체결가.
⚠ 적정 Swap Points가 얼마인지 모름. 은행만 이론가를 알고 있음.
⚠ 연간 누적하면 상당한 비용 차이인데, 이를 인지조차 못함.
⚡ Step 1 — StableFX가 적정가격을 먼저 보여줌

연간 효과
주요 연구에 따르면, 복수 은행 간 경쟁 구도를 만드는 것만으로 최대 20bps까지 거래 비용 절감 가능.
연간 USD $50M 거래 기업 기준, 최대 ₩1.4억 절감에 달하는 금액입니다.

2. 은행별 전화/메신저로 개별 요청. 시간 소요.
3. 호가 비교를 엑셀에 수기 정리
적정가 인지는 됐지만 경쟁 호가 확보는 수동

✓ 재협상 시 은행명 자동 마스킹 — 관계 유지
✓ 거래별 은행 마진 축적 → 분기 리뷰 자동 생성
AI가 RFQ 발송 → 최저 호가 은행 안내 → 마스킹 재협상 → 승인만

AI Agent Modes

당신의 AI Treasury, 어떤 성향으로 운용하시겠습니까?

테슬라가 운전 모드를 선택하듯, 우리도 AI Agent의 운용 성향을 설정합니다.
CONSERVATIVE
🐢 유동성 수호형
• 잔고 부족 예방이 최우선
• 이자 수익보다 유동성 확보 중시
• 리밸런싱 시 항상 여유 버퍼 유지
• 헷지 비율을 보수적으로 높게 유지
"현금이 왕이다. 절대 부족하면 안 된다."
STANDARD
⚖️ StableFX 기준형
• StableFX 이론가 기준으로 성실 처리
• 들어오는 건에 대해 적정가 비교 후 체결
• 스케줄대로 헷지, 리밸런싱 실행
• 편향 없이 데이터 기반 판단
"이론가대로, 스케줄대로, 빈틈없이."
AGGRESSIVE
🔥 초과수익 추구형
• 시장 이벤트(NFP 등) 적극 활용
• 유리한 타이밍에 선제적 체결
• 분할 매매로 평균 단가 최적화
• 이자율 차이 적극 활용한 리밸런싱
"리스크를 감수하더라도 최대 수익을."
🛠 우리 회사에 맞게 커스터마이징
각 모드는 회사의 Risk Tolerance에 따라 세부 파라미터를 조정할 수 있습니다.
• 헷지 비율 목표: 50% ~ 90% 범위 설정
• 최대 단건 거래 금액 한도
• 이벤트 대응 자동화 수준 (알림만 / 제안 / 자동 실행)

PLG 무료 유입 → 유료 전환 → 기업 도입. 이론가 계산 → 헷지 안내 → 자금 자동 재배치까지 한 사이클 완성.
AG-01 · PLG 무료
Market Data Feed
환율(spot), 금리커브, 스왑포인트(tenor별) 실시간 수집. AG-07의 연료.
AG-02 · PLG 무료
Funding Cost
고객별 통화별 effective rate 역산. 범용 계산기를 우리 회사 맞춤으로 전환하는 핵심.
AG-07 · PLG 무료
FX Pricing Engine
이론가 계산 + 은행 마진(bps) 시각화. PLG의 문. 이게 없으면 고객이 안 들어옴.
AG-08 · 유료
Hedge Strategy
Forward vs Swap, 만기 구조, 헷지 비율별 비용-효과 시뮬레이션. 뉴스 감지 + 유사 사건 분석.
AG-06 · 유료
Liquidity Hub
Balance+Schedule 기반 계좌 과부족 예측. 통화별 Effective Rate 분석 후 저금리 계좌에서 자동 재배치.
SEAL-01 · 기업 필수
Policy & Approval
RBAC/SoD 기반 승인 워크플로우, 권한 통제. 기업 도입의 전제조건.

Roadmap

도입 로드맵

무료 진입 → 가치 확인 → 기업 전사 도입. 각 단계가 다음 단계를 자연스럽게 유도합니다.
PHASE 1
PLG 무료 진입
StableFX 이론가 계산기
• 은행 호가 vs 이론가 비교
• 은행 마진(bps) 실시간 확인
• Effective Rate 계산기
가입만 하면 즉시 사용. “우리가 은행에 얼마나 더 내고 있었는지” 깨달음.
→
PHASE 2
TMS + AI Agent
통합 자금관리 + 자동화
• Cash Balance / Schedule 통합
• Trade Input → Blotter 자동 반영
• AI 이상 탐지 + 리밸런싱
• 헷지 전략 안내 + 뉴스 분석
이론가를 아는 순간, 거래 관리 전체를 원하게 됨.
→
PHASE 3
Multi-bank RFQ
SEAL 기반 완전 실행
• 다수 은행 동시 RFQ + 마스킹
• SEAL 승인 워크플로우
• 자동 리포트 + 감사 추적
• CFO 대시보드 + 의사결정 지원
TMS의 모든 데이터가 거래소 인프라를 사전 검증.
